How scoring works
This tool analyzes the XML structure of a Tableau workbook (.twb/.twbx) and scores it on a 1–10 scale representing how far it has deviated from the BIC standard, where 1 = pure data repoint and 10 = net-new solution.
Scale definition
Score Band Description
1–2Automatic BICPure repoint or branding only — no customization effort
3–4Auto+Minor layout/viz changes or new visuals with existing fields
5–6EvaluateRedesigned dashboards or logic using existing BIC fields
7–8Custom buildClient-specific fields, KPIs, or logic requiring new semantic fields
9–10Do not use BICNear full rebuild or net-new solution
Signals used
Entity/domain fields (50%)
Remote fields not in any confirmed BIC build — client-specific business data like parking, security_deposit
Missing BIC areas (20%)
BIC content areas absent from the workbook — indicates structural divergence
Custom dashboards (15%)
Dashboards added beyond any known BIC version
Metric extensions (5%)
BIC-style calculated metrics not in standard — lower weight as common add-ons
WS density / Design (10%)
Worksheet-per-dashboard ratio vs standard; non-CBRE color palettes

Privacy & data handling
All processing happens locally in your browser. No workbook data, field names, or results are sent to any server or external service. The tool reads only the XML structure of .twbx files — dashboard names, field names, and metadata. No actual lease data or client data is read.
